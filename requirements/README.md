# Requirements

- `core-platform.md` covers contract and service-boundary requirements.
- `backlog-staging.md` defines the staged delivery slices used by the initial backlog.
- `scenario-simulation.md` covers scenario-pack and simulator requirements.
- `infrastructure.md` covers infra and GitOps requirements.
- `replacement-proof.md` covers the conformance and swap-demo requirements.

The requirement set is intentionally compact for the initial scaffold. It defines what must stay true while implementation fills in the gaps.
