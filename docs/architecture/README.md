# Architecture

- `platform-architecture.md` defines the target runtime shape for the kernel, simulator, and platform layers.
- `data-architecture.md` defines service-owned persistence, integration patterns, and cross-service data rules.
- `implementation-plan.md` turns the architecture into a staged delivery plan aligned with the current backlog.

The ADR under `../adr/0001-contract-first-platform.md` remains the root architectural constraint. These documents expand it into implementation guidance without weakening the service boundaries.
